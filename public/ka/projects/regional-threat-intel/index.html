რეგიონალური საფრთხეების დაზვერვის აგრეგატორი

November 27, 2025 ·2 წთ კითხვა ·CTI Python Grafana

PythonPostgreSQLGrafana

მიმოხილვა

ეს მორგებული CTI დეშბორდი წყვეტს ინფრომაციული გადატვირთვის პრობლემას. სისტემა ფილტრავს დიდი მოცულობის ღია წყაროებს და მათგან აგენერირებს მაღალი სიზუსტის ინდიკატორების სიას. პლატფორმა ფოკუსირებულია მხოლოდ განსაზღვრული საფრთხეების გარემოსთვის რელევანტურ APT აქტივობებსა და C2 ინფრასტრუქტურაზე თვალის დევნებისთვის.

1. ტექნიკური არქიტექტურა და მონაცემთა ნაკადის მიღება

სისტემა მუშაობს კონტეინერიზებულ მიკროსერვისულ არქიტექტურაზე მედეგობისა და მარტივი მართვის უზრუნველსაყოფად:

  • Python-ზე ბაზირებული კოლექტორები მუშაობენ როგორც API მომხმარებლები და მონაცემთა პარსერები.
  • PostgreSQL უზრუნველყოფს სტრუქტურირებული IOC-ების საიმედო რელაციურ შენახვას.
  • Grafana გამოიყენება რეალურ დროში მონიტორინგისა და საფრთხეების გარემოს ვიზუალიზაციისთვის.

მონაცემთა აგრეგაცია და ფილტრაცია

დამოუკიდებელი კოლექტორები მართავენ მონაცემთა მიღებასა და შეფასებას:

  1. collector-news: კითხულობს RSS ფიდებს და აფასებს სტატიებს მაღალი რელევანტურობის მქონე საკვანძო სიტყვების საფუძველზე, როგორიცაა კრიტიკული ინფრასტრუქტურა, კიბერშეტევა ან მოწყვლადობის გამჟღავნება, ასევე იყენებს რეგიონალურ ფილტრებს.
  2. collector-ioc-threatfox / collector-otx: იღებს ტექნიკურ ინდიკატორებს, მათ შორის IP მისამართებსა და ჰეშებს ღია წყაროებიდან. მონაცემთა ბაზაში ჩაწერამდე სისტემა მათზე გეოგრაფიულ და კონკრეტულ კამპანიებზე დაფუძნებულ ფილტრებს ატარებს.

2. დაზვერვის ოპერაციონალიზაცია

სისტემის მთავარი ღირებულება არის მისი ინტეგრაცია დაცვის ინსტრუმენტებთან:

  • მონაცემთა ბაზისთვის იქმნება ცალკეული, მხოლოდ წაკითხვის უფლების მქონე PostgreSQL მომხმარებელი.
  • ეს საშუალებას აძლევს უსაფრთხოების გუნდს, რომ ყოველ საათში პირდაპირ CTI ბაზიდან წამოიღონ გაფილტრული, მაღალი სიზუსტის ბლოკლისტი და ავტომატიზაცია გაუკეთონ დაცვის ციკლს.

3. დეპლოი და გარემოს კონფიგურაცია

სრული სტეკი გაშვებულია Docker Compose-ის მეშვეობით პორტაბელურობისა და სწრაფი გადაწყობის გარანტიისთვის.

  • სისტემა განთავსებულია Nginx Reverse Proxy-ს უკან SSL/TLS სერტიფიკატით Grafana-ს ვებ ინტერფეისის დასაცავად.
  • ავტორიზაციის მონაცემების გაჟონვის თავიდან ასაცილებლად, Grafana უკავშირდება შიდა postgres:5432 კონტეინერს გარემოს ცვლადებისა და გაზიარებული შიდა ქსელის გამოყენებით.

მოცემული მაგალითი აჩვენებს კონფიგურაციას, რომელიც საჭიროა Grafana-ს ვებ ინტერფეისის დასაცავად და შიდა მონაცემთა ბაზასთან დასაკავშირებლად.

grafana:
    # ... (დეტალები გამტოვილია)
    environment:
      - GF_SECURITY_ADMIN_USER=${GF_ADMIN_USER}
      # მონ.ბაზა იყენებს დოკერის შიდა ქსელის სახელს
      - GF_SERVER_ROOT_URL=https://IP_OR_DOMAIN/  
    networks:
      - cti_network
    depends_on:
      - postgres

შენიშვნა: კონკრეტული რეგიონალური ფილტრაციის საკვანძო სიტყვები, API გასაღებები და პროდაქშენის სენსიტიური კონფიგურაციის დეტალები გამოტოვებულია. ეს ანგარიში ფოკუსირებულია არქიტექტურულ გადაწყვეტილებებსა და ფილტრაციის მეთოდოლოგიაზე, რომელიც გამოიყენება დიდი მოცულობის CTI მონაცემების პრობლემის მოსაგვარებლად.
