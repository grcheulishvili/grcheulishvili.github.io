Mantis C2 არქიტექტურა

November 27, 2025 ·2 წთ კითხვა ·MalDev Go C2

მიმოხილვა

Mantis C2 არის სატესტო, საგანმანათლებლოდ შექმნილი Command and Control ფრეიმვორკი, რომელიც შექმნილია EDR ტელემეტრიის გასატესტად და მისთვის გვერდის ავლისთვის. კომერციული და ფართოდ ხელმისაწვდომი C2 ფრეიმვორკები ხშირად შეიცავენ მკაცრად სიგნატურებულ სტაბებს (stubs); Mantis აწყობილია ნულიდან, რათა მინიმუმამდე დაიყვანოს სტატიკური და ქცევითი ანომალიების კვალი.

1. გვერდის ავლისა და პერსისტენტობის არქიტექტურა

AST-ზე დაფუძნებული პოლიმორფიზმის პროტოკოლი

სტატიკური ანალიზის დასამარცხებლად, ბილდერი იყენებს კოდის მუტაციის მორგებულ პროტოკოლს:

  • აგენტი მუშავდება სპეციალური პრე-კომპილატორის მიერ, რომელიც ცვლის სინტაქსურ განშტოებებს (AST) ყოველი დეპლოისთვის სტრუქტურულად უნიკალური ბინარული ფაილების დასაგენერირებლად. ეს ამარცხებს CFG ანუ კოდის სტრუქტურის ანალიზსა და სტატიკურ სიგნატურებს, რაც უბრალოდ ფაილის ჰეშის მუტაციაზე ბევრად წინ გადადგმული ნაბიჯია.
  • ყველა სენსიტიური სტრიქონები (მაგ. C2 დომენები, ბრძანებები) ავტომატურად იძებნება და იშიფრება კოდის დონეზე კომპილაციამდე, ხოლო მათი გაშიფვრა ხდება runtime-ის დროს, ფაილის მეხსიერებაში დინამიურად ჩატვირთული გასაღების მეშვეობით.
  • შემდეგ ეტაპზე, დისკზე ჩაწერის ნაცვლად, Mantis მეხსიერებას გამოყოფს დინამიურად და პირდაპირ უშვებს შელკოდს.
  • კროს-კომპილაციის დროს ავტომატურად იშლება DWARF ცხრილები და სიმბოლური ინფორმაცია, რაც მნიშვნელოვნად ართულებს რევერსინგის პროცესს.

სიჩუმე

  • აგენტი ფარულად მუშაობს ფონურ პროცესად.
  • სერვერთან კავშირის ინტერვალები მაქსიმალურად რანდომიზებულია. გამოღვიძების მომენტში აგენტი ნიღბავს თავის მეხსიერების რეგიონებს, რათა თავიდან აიცილოს არააქტიური მავნე thread-ების აღმოჩენა მეხსიერების სკანირებისას.

თვითგანადგურება

წინასწარ განსაზღვრული კოდური სიტყვით განსაზღვრული ბრძანება ააქტიურებს თვითგანადგურების რუტინას, რაც კრიტიკულია ოპერატიული დასუფთავებისთვის:

  1. შლის Windows Registry Run გასაღებებს, რომლებიც გამოიყენებოდა პერსისტენტობისთვის.
  2. შლის საკუთარ ბინარულ ფაილს დისკიდან.
  3. წყვეტს სესიას და აგზავნის სიგნალს C2 სერვერზე.

2. მონაცემთა ტელემეტრია

მთავარი სიძლიერე არის დაყოფილი ტელემეტრიის მოდელი, რომელიც შექმნილია დიდი მოცულობის მონაცემების სწრაფი ექსფილტრაციისთვის:

  1. მონაცემები მყისიერად იგზავნება მცირე, დაშიფრული პაკეტების სახით ფარული TCP არხის მეშვეობით. ეს უზრუნველყოფს მონაცემთა უწყვეტ მიღებას იმ შემთხვევაშიც კი, თუ სესია მოულოდნელად შეწყდება.
  2. კავშირის დამყარებისთანავე გენერირდება მყისიერი Stage 1 Recon Report, რომელიც ოპერატორს აწვდის კრიტიკულ სისტემურ და ფაილურ მეტამონაცემებს ნებისმიერი ბრძანების გაცემამდე.
  3. ფაილები, რომლებიც აჭარბებენ განსაზღვრულ ლიმიტს, ავტომატურად გადაიცემა ნაწილ-ნაწილ, რაც გვერდს უვლის კომპრესიის ძრავს მეხსიერების გადატვირთვის თავიდან ასაცილებლად და CPU-ს მინიმალური დატვირთვის შესანარჩუნებლად.

3. ბილდერი და სანიტიზაცია

ფრეიმვორკი შეიცავს პეილოდების დასაგენერირებელ ინსტრუმენტს.

შენიშვნა: უსაფრთხოების სტანდარტების დაცვისა და ბოროტად გამოყენების თავიდან აცილების მიზნით, ეს საჯარო დოკუმენტაცია არ მოიცავს ისეთ ძირითად ფუნქციებს, როგორიცაა შიფრაციის გასაღებები, C2 IP/პორტის კონფიგურაცია და მრავალნაკადიანი სესიების მართვის სრული ლოგიკა. პოლიმორფული ბილდის უკან არსებული ტექნიკური მექანიზმების გასაცნობად, იხილეთ შესაბამისი კვლევა: Go AST პოლიმორფული ძრავი.

ეს ინსტრუმენტი განკუთვნილია მხოლოდ საგანმანათლებლო კვლევებისა და ავტორიზებული შიდა საფრთხეების სიმულაციისთვის.
